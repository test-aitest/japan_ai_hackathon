# Requirements Definition (requirements.md)

## 1. プロジェクト概要

ブラウザ上で動作する、リアルタイム AI 音声翻訳表示アプリケーション。
ユーザーが話した言葉を即座に認識し、指定された言語に翻訳してテキストとして画面に表示する。
データベースや認証は不要。読み上げ機能は持たず、字幕やチャットログのように翻訳結果を可視化することに特化する。

## 2. ユーザーストーリー

- **ユーザー:** 言語の異なる会議に参加している人、または翻訳字幕が欲しい人。
- **ゴール:** 会話の内容をリアルタイムで翻訳し、テキストで確認することで理解を深める。

## 3. 機能要件リスト

### 3.1 翻訳設定 (Translation Setup)

- **言語選択:**
  - **Source Language (話す言語):** 日本語、英語、中国語などから選択。
  - **Target Language (訳す言語):** 翻訳先の言語を選択。

### 3.2 リアルタイム翻訳セッション (Real-time Translation)

- **音声認識 (STT):**
  - AssemblyAI を使用。選択された「Source Language」に合わせて認識言語を設定する。
  - リアルタイムで流れてくる暫定的な文字起こし結果（Partial Transcript）と、確定結果（Final Transcript）をスムーズに表示する。
- **AI 翻訳 (LLM):**
  - 文字起こしが確定したタイミングで OpenRouter (LLM) にテキストを送信。
  - 入力テキストを指定言語へ翻訳し、結果を表示する。
- **UI 表示:**
  - **翻訳ログ:** 左側に原文、右側に翻訳文を表示するリスト形式。
  - **ステータス表示:** 「聞き取り中...」「翻訳中...」などの状態を可視化する。

### 3.3 履歴操作

- リロードまたは「Reset」ボタンで表示されている翻訳履歴を消去する。

## 4. 非機能要件

- **ステートレス:** サーバーにログを保存しない。
- **リアルタイム性:** 文字起こしから翻訳表示までの遅延を極力短くする。
